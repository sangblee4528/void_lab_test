# Git 및 GitHub 사용 가이드

이 문서는 변경 사항을 확인하고 효율적으로 커밋하여 GitHub에 업로드하는 방법을 설명합니다.

## 1. 변경 사항 확인 및 일괄 추가

파일을 하나씩 추가하는 대신, 변경된 파일 목록을 확인하고 한꺼번에 스테이징 영역에 추가하는 방법입니다.

### 🔍 현재 상태 확인 (변경된 파일 리스트 보기)
어떤 파일들이 수정되었는지 전체 리스트를 확인합니다.
```bash
git status
```

### ➕ 변경된 모든 파일 추가 (일괄 스테이징)
수정된 모든 파일을 한 번에 추가합니다. `.`은 현재 디렉토리와 하위 디렉토리의 모든 변경 사항을 의미합니다.
```bash
git add .
```
> [!TIP]
> 특정 디렉토리의 변경 사항만 추가하고 싶다면 `git add docs/` 와 같이 경로를 지정할 수 있습니다.

## 2. 변경 사항 커밋

스테이징된 파일들에 대해 변경 내용을 설명하는 메시지와 함께 기록을 남깁니다.
```bash
git commit -m "커밋 메시지 입력 (예: docs: 질문_native_흐름도 업데이트)"
```

## 3. GitHub에 업로드 (Push)

로컬의 커밋 내역을 원격 저장소(GitHub)로 전송합니다.
```bash
git push origin main
```

---

## 💡 요약 프로세스 (한눈에 보기)

가장 많이 사용하는 핵심 명령어 조합입니다:

```bash
git status    # 1. 변경 리스트 확인
git add .      # 2. 모든 변경 사항 추가
git commit -m "작업 내용"  # 3. 확정
git push origin main    # 4. 서버 전송
```

## 🛠️ 고급 복구 및 이력 확인 방법 (Problem Solving)

문서나 코드를 수정하다가 과거의 특정 내용이 유실되었거나, 누가 무엇을 바꿨는지 확인해야 할 때 유용한 기능들입니다.

### 0️⃣ 과거 커밋 ID(해시) 찾는 방법
복구 명령어를 쓰려면 먼저 "어느 시점"으로 돌아갈지 알아야 합니다.
```bash
# 전체 요약 기록 보기
git log --oneline

# 특정 파일의 수정 기록만 보기 (ID 찾기 가장 쉬운 방법)
git log --oneline docs/질문_native_loop_흐름도.md
```
*   맨 왼쪽의 **7자리 영숫자(예: a7ac589)**가 바로 커밋 ID입니다.

### 1️⃣ 특정 파일의 상세 변경 이력 확인 (`git log -p`)
파일의 각 커밋별로 어떤 내용이 추가(+)되고 삭제(-)되었는지 코드 레벨에서 확인합니다.
```bash
git log -p docs/질문_native_loop_흐름도.md
```
*   `j / k`: 아래/위로 이동
*   `/검색어`: 특정 텍스트 찾기
*   `q`: 종료

### 2️⃣ 삭제된 내용(키워드)으로 커밋 찾기 (`git log -S`)
파일 이름이 기억나지 않아도, 유실된 특정 "단어"나 "코드"가 포함되었던 시점을 찾을 수 있습니다.
```bash
# "agent_proxy_server.py"라는 글자가 포함된 적이 있는 모든 커밋 찾기
git log -S "agent_proxy_server.py" --all --oneline
```

### 3️⃣ 특정 시점의 파일 내용 통째로 보기 (`git show`)
과거 커밋 당시의 파일 상태를 그대로 화면에 출력합니다. 특정 스니펫을 복사해올 때 유용합니다.
```bash
# 구문: git show 커밋ID:파일경로
git show a7ac589:docs/질문_native_loop_흐름도.md
```
*   가장 최근 커밋의 직전 상태를 보려면: `git show HEAD~1:경로/파일명`

### 4️⃣ 파일 복구 및 취소 (`git restore`)
커밋된 적이 있는 파일이라면 Git을 통해 쉽게 복구할 수 있습니다.
```bash
# 특정 파일 복구
git restore docs/질문_native_loop_흐름도.md

# 모든 변경 사항 취소 (마지막 커밋 상태로 일괄 복구)
git restore .
```

### 5️⃣ 서버에서 최신 상태 다시 받기 (`git pull`)
로컬에서 파일을 지우고 커밋까지 완료한 경우, GitHub 서버에 있는 원본을 다시 가져옵니다.
```bash
git pull origin main
```
